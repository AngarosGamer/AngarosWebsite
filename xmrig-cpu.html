Use Salad in your command line with XMRig CPU

Made by Angaros#1263 - Last update 19/06/2022 @15:19 UTC+1

Running CLI allows you to:

  • Custom limit CPU usage - By percentage and/or on a per-core basis
  • Set max temperatures and start temperatures
  • Potentially earn a bit more by using more advanced miner arguments than what Salad uses

Depending on your local electricity costs and your CPU's performance, CPU mining may potentially not be profitable for you.   • Internet Connection
  • A supported CPU - At least 2MB of L3 CPU cache
  • 5GB+ RAM
  • 500MB disk space
  • Have installed Salad (0.5.6 if you wish to use Nicehash's pool, rather than Prohashing.)
  • To have mined once before (This is to generate your Rig and wallet ID, shown in Salad's logs in later steps)

Start by installing the latest XMRig gcc-win64.zip version, found Here. This will install the base XMRig miner with the dependencies and required files. It is a rather small package, so it shouldn't take too long. Extract it to your folder of choice.

Here is a guide to finding your Salad logs. You'll need these to find the required IDs.

Note that Salad 1.0 no longer gives the required IDs to mine with Nicehash in the logs - If you wish to use Nicehash's pool, you'll have to use Salad 0.5.6 to get the IDs required for these pools.

You may be able to still get your Prohashing IDs in Salad 1.0 by mining at least once with it, heading to your Salad logs folder, opening the folder for one of the miners (e.g. XMRig), opening the latest log inside said folder, then at the very top you should see something like "-u salad -p (A bunch of random characters)".

It should look something like this:

You'll want to keep these for when we do the linking

- Go in the XMRig folder you installed in Step 1. You'll find a file named start.bat. You'll want to open it with notepad, or a code editor.

Should it not exist, you can also just create it.

- We will need to edit the start file, by adding the correct information:

Nicehash:

@echo off
xmrig.exe --donate-level=1 -o <pool> -u <Wallet Address>.<Rig ID> -a randomx -k --nicehash
pause

Prohashing:

@echo off
xmrig.exe --donate-level=1 -o <pool> -u salad -p <That long string of text> -a randomx -k
pause

- First, the pool and port: You'll want to use one of these two:

  • stratum+ssl://randomxmonero.auto.nicehash.com:443 - Nicehash RandomX. If you're having issues with the SSL pool, you can alternatively use stratum+tcp://randomxmonero.auto.nicehash.com:9200
  • stratum+tcp://prohashing.com:3359 - Prohashing RandomX. Be sure to have ,a=randomx at the end of your 'long string of text'.

- And lastly, the Wallet & Rig IDs: This is pretty straightforward. Add them to the file like shown in the example. You'll need to use the Nicehash values for 'Wallet Address' and 'Rig ID'.
For Prohashing, you'll need to use your 'long string of text' Prohashing ID. If the algorithm indicator at the end of the ID (,a=) doesn't work, try removing it. It should work without it.

- Here's an example of a completed file, using Nicehash's pool:

@echo off
xmrig.exe --donate-level=1 -o stratum+ssl://randomxmonero.auto.nicehash.com:443 -u 373Ee6DSDgAYNAVSnfgUV99248L6Ab7aUH.p36pcuzwyhvd2fx -a randomx -k --nicehash
pause

- And here's an example of a completed file, using Prohashing's pool:

@echo off
xmrig.exe --donate-level=1 -o stratum+tcp://prohashing.com:3359 -u salad -p o=b256ea13-797f-474f-b6c9-c8edd47d9787,n=b256ea13-797f-474f-b6c9-c8edd47d9787,a=randomx -a randomx -k
pause

Here's some pretty useful XMRig commands you can use:

  • --threads=N : Limits CPU usage using threads (If you have 8 threads, setting N to 4 will mean the miner can only use 4 out of 8 threads; effectively 50% usage)
  • --print-time=n : If you want to print (update) hashrate at a faster rate, "n" being in seconds
  • -k : (Keepalive) Will send packets to pool to prevent timeout
  • --background : Will run XMRig in the background. Note: Will not appear in the System Tray, you'll have to close it via Task Manager
  • --cpu-priority=n : Where 'n' is a number between 0 - 5, determining the miner's process priority. 0 is for idle, 2 is normal priority (Default), and 5 is the highest priority. Usually leaving this as default is good enough, but if you don't mind your PC being slower, you can manually set this higher.

There are more to be found here for additional customisation.

- Here's an example of a completed file, using Nicehash's pool with extra commands:

@echo off
xmrig.exe --donate-level=1 -o stratum+ssl://randomxmonero.auto.nicehash.com:443 -u 373Ee6DSDgAYNAVSnfgUV99248L6Ab7aUH.p36pcuzwyhvd2fx -a randomx -k --nicehash --print-time=5 -k
pause

Save the start.bat file, close it, and run it. It should work if everything was set up properly.

Wait about 15 minutes (can be a bit more, a bit less) and verify that you are getting accepted shares, and then earning on Salad. If not, something went wrong. Verify you followed all steps and retry once the issue is fixed.

You can optionally choose to run XMRig as administrator to enable 'MSR', which you can read more about Here.

The above article should do a good job of explaining what it is, but it basically optimises some settings on the OS level, noticeably increasing your hashrate as a result.

Typically you should be cautious about giving miners administrative privileges, and you'd be right for doing so!
XMRig is, however, one of the few exceptions to this rule, because it is entirely open source.
This means that its many hundreds of thousands of users can easily dive into XMRig's source code and verify for themselves that there is indeed nothing shady going on.


To do this, you'd want to right-click on xmrig.exe, then click on Properties.

You next click on the 'Compatibility' tab at the top.

Near the bottom, tick 'Run this program as administrator'.

Hit apply, then run your start.bat file, and XMRig should now start as administrator and automatically apply the MSR mod!
