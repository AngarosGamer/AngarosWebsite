Use Salad in your command line with PhoenixMiner

Made by Angaros#1263 - Last update 15/06/2022 @18:41 UTC+2

Running CLI allows you to:

  • Choose custom GPU usage
  • Set fan min / max
  • Set max temperatures and start temperatures
  • Potentially earn a bit more

  • Internet Connection
  • A dedicated GPU with 3GB+ VRAM
  • 500MB disk space
  • Have installed Salad (0.5.8 if you wish to use Nicehash or Ethermine pools)
  • To have mined once before (This is to generate your Rig and wallet ID, shown in Salad's logs in later steps)

You'll first want to find PhoenixMiner from your Salad files. This can be done, in windows, by searching for %appdata%, and opening that folder. Next, navigate to the plugin-bin folder, and open the latest PhoenixMiner version you can find. For the rest of this guide, we'll be doing our operations in that folder.

Should you not find it, or followed this guide specifically because it didn't work on Salad, you can find the download of the folder over here (the zip file)

You would also want to whitelist the folder that you place PhoenixMiner in, just to ensure that your antivirus doesn't decide to interfere.

Here is a guide to finding your Salad logs. You'll need these to find the required IDs.

Note that Salad 1.0 no longer gives the required IDs to mine with Nicehash and Ethermine in the logs - If you wish to use either of these pools, you'll have to use Salad 0.5.8 to get the IDs required for these pools.

You may be able to still get your Prohashing IDs in Salad 1.0 by mining at least once with it, heading to your Salad logs folder, opening the folder for one of the miners (e.g. T-Rex), opening the latest log inside of said folder, then at the very top you should see something like "-u salad -p (A bunch of random characters)".

- Go in the PhoenixMiner folder you installed in Step 1. You'll want to make a new file named start.bat (or any name you want, I just chose start for conveniency). To do that, create any text file (right click --> "New" --> "Text Document"), then save it as start.bat

- And next you'll want to start editing it.

- Enter the following two lines:
PhoenixMiner.exe -pool <pool> -wal <Wallet ID>.<Rig ID>
pause

  • The pause just ensures that, if the miner closes unexpectedly, the CMD window will stay open so you can troubleshoot easily.


    • - First, the pool and port:

    • If you have a 4GB VRAM GPU, you can mine either Ethereum Classic, or Ravencoin:
      eu1-etc.ethermine.org:4444 - For Ethermine. You can optionally change locations to the one closest to you: eu1, us1, asia1.
      stratum+ssl://kawpow.auto.nicehash.com:443 - For Nicehash. If you have issues with SSL, you can alternatively use stratum+tcp://kawpow.auto.nicehash.com:9200

    • If you have a 6GB+ VRAM GPU, you should be mining Ethereum:
      eu1.ethermine.org:4444 - For Ethermine. You can change locations: eu1, asia1, us1, us2, asia2
      stratum+ssl://daggerhashimoto.auto.nicehash.com:443 - For Nicehash. If you have issues with SSL, you can alternatively use stratum+tcp://daggerhashimoto.auto.nicehash.com:9200

    • Our knowledge for Prohashing is limited, due to it being relatively new...
      stratum+tcp://prohashing.com:3339 should be the correct pool for Ethash (Ethereum).
      stratum+tcp://prohashing.com:3361 should be the correct pool for KawPow (Ravencoin).

      Your Wallet ID for Prohashing would be -wal salad and instead of a Rig ID, you'd use the separate argument -pass {That long string of text}


    - Second, the wallet and rig. You just have to replace <Wallet ID> and <Rig ID> with the values you have in your logs.

    If you do not replace the Wallet and Worker ID with your own IDs in the -wal option, you won't be getting any earnings.


    For Prohashing, Ensure that the end of your 'really long text' has the correct algorithm! For Ethash it should be ,a=ethash, and for KawPow it should be ,a=kawpow. Though if this doesn't work, try removing the ',a' algorithm indicator. It should work without it.

    The final result of the bat file should look like, for a 6GB+ VRAM GPU:

    PhoenixMiner.exe -pool stratum+ssl://daggerhashimoto.auto.nicehash.com:443 -wal 33kJvAUL3Na2ifFDGmUPsZLTyDUBGZLhAi.8sqe3di5hfeeonq
    pause

    So far you've got the minimum to be able to mine, though if you don't want to just stop there, there are tons of customization options you can add to change your mining experience. Here are a few of the most useful commands:

    • -gpow <percent> where percent is the maximum percentage you want your GPU to mine at. If you want the miner to use only 50% of your GPU, type -gpow 50. People mostly favour reducing their power limit in overclocking software, though this does still have its uses.
    • -powlim <n> where n is the power limit you want (between -75 to 75)
    • -cclock <MHz> For AMD, enter a core clock in MHz. For NVIDIA, use relative values (e.g. -300 or +400)
    • -mclock <MHz> sets GPU memory in MHz
    • -fanmin <percent> where percent is the percentage you want your fans to run at MINIMUM (if you want them to always run somewhere above 70%, type -fanmin 70)
    • -fanmax <percent> same as above, but determines a limit to max fan speeds.
    • -tstop <degrees> where degrees is, in celsius, the temperature at which your GPU will stop if it reaches it. It will stop once it reaches that temperature (if you don't want your GPU to go above 70 °C, type -tstop 70)
    • -tstart <degrees> where degress is, in celsius, the temperature at which your GPU will start if it reaches it. It will start once it falls to that temperature (if you want to start your GPU when it reaches 50 °C, type -tstart 50)

    You'll usually want to use tstop and tstart together, as once started it won't stop, and once started it won't stop unless started. Aka, if it reaches tstop limit, it will stop for always unless there is a tstart to get it working again at some point.

    General info for the additional commands:

    • You'll want tstop to be higher than tstart.
    • You'll want fanmin to be smaller than fanmax.
    • You'll want possible temperatures for tstart and tstop. (say, if your GPU is unable to go below 50°C, don't set a tstart at 30°C, otherwise it will never happen, same for tstop.)
